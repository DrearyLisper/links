# Links Vault — Claude Context

## What this is
A personal link vault stored in a single `README.md`, organized by topic sections, viewable on GitHub.

## Workflow when the user adds a link
1. Fetch the page to get the title and a short description
2. Determine the category — add to existing H2 section or create a new one
3. Add the entry in this format:
   ```
   - [Title](URL) — Short description. `tag1` `tag2` *(added: YYYY-MM-DD)*
   ```
4. Add 2–3 **See also** links as sub-bullets

## See also rules
- Only specific articles, blog posts, essays, books, or videos
- Never: aggregators, code repositories, wikis, newsletters, or general-purpose sites
- Use WebSearch to find fresh (2025–present) See also links when possible — prioritize recency
- WebSearch is for **See also only**, never for finding the main entry (user always provides that URL)

## README.md structure
- One H2 per category, sorted alphabetically
- Bullet list of entries under each H2
- See also as indented sub-bullets under each entry
